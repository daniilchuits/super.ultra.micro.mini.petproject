# File Processing Service

Сервис обрабатывает текстовые файлы, сохраняет результат в POSTGRESQL. Написан на Go. Запускается с помощью docker compose.

## Возможности
- создание задач для дальнейшего выполнения 
- асинхронная обработка worker'ом
- хранение результатов в POSTGRESQL
- аутентификация с помощью JWT-token'а
- запуск с помощью Docker Compose 
- логирование ошибок

## Архитектура
```text
Client
  |
  v
Gateway
  |
  +--> Auth Service
  |       |
  |       v
  |   PostgreSQL
  |
  +--> Worker Service
          |
          v
      PostgreSQL
```

## Запуск
1. Клонировать репозиторий `git clone <repo-url> <your-path>`
2. Перейти в директорию проекта `cd <your-path>`
3. Запустить `docker compose up --build`

Переменные окружения используются только в docker-compose.yaml, вот они:
```text
POSTGRES_USER=...
POSTGRES_DB=...
POSTGRES_PASSWORD=...
POSTGRES_USER_FOR_AUTH=...
POSTGRES_DB_FOR_AUTH=...
POSTGRES_PASSWORD_FOR_AUTH=...
JWT_SECRET=...
```
## API
порт используется 8080

### регистрация и логин
регистрация - POST http://localhost:8080/auth/register
логин POST http://localhost:8080/auth/login

Пример тела запроса для регистрации и логина
{
    "login": "user123", (логин >= 6 символов, больше валидации нет)
    "password": "123456" (пароль >= 6 символов)
}

В ответе при успешном логине, получаешь jwt-token. Он длится 2 часа и его необходимо прикрепить к запросу в headers. 
Пример заголовка:
```http
Authorization: Bearer <jwt-token>
```

После этого к каждому запросу в заголовки устанавливается токен.
Токен длится 2 часа!
В токене содержится время, до которого он валиден и ID пользователя, которое берется из таблицы со всеми регистрациями.

Пользователь может просматривать только свои записи, при занесении каждой в таблицу, присваивается user_id для этой записи.

## Обработка

Получение данных по всем загруженым файлам пользователя - GET http://localhost:8080/work/jobs

Получение данных по конкретному файлу - GET http://localhost:8080/work/jobs/{id записи}

Публикация файла (по сути только публикация названия) - POST http://localhost:8080/work/jobs
Пример тела запроса:
{
    "filename": "zxc.txt"
}
Файлы принимаются только в `.txt` формате.

Публикация файла - POST http://localhost:8080/work/upload
Присылать файл через postman можно через тело запроса:
    Body->form-data выбираешь тип (File)))), ключ (по которому код и получает файл) = file и value - выбираешь файл из локального хранилища. Отправляешь запрос и статус файла меняется с 'pending' на 'processing'.
Раз в 5 секунд параллельно коду, файлы со статусом 'processing' в случайном порядке забирает worker. Файл предварительно был скопирован локально с помощью volume, чтобы не быть утерянным при остановке контейнера. Файл обрабатывается и результатом его обработки обновляется соответствующая запись в postgresql.
